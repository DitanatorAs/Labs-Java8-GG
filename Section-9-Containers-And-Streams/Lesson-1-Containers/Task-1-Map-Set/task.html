Task-1-Map-Set:

  1. Ориентируясь на материал видеолекции видоизмените код ClassMap из раздела Exceptions-IO (где переделывали на выброс исключения ClassMapException) так, чтобы использовать контейнер Map для хранения размещения учеников вместо двумерного массива desks.
    Просто измените тип переменной desks на Map и далее уже меняйте реализацию в методах.

    Интерфейс класса ClassMap должен остаться абсолютно таким же как в Exceptions-IO кроме метода getDesks так как он теперь должен возвращать Map.
    Меняется только внутренняя (а бывает внешняя? :)) реализация.

    Считаем, что имена учеников уникальные. Размещение ученика кодируйте с помощь строки place="столбец_ряд". Размещайте ученика в контейнере следующим образом:
    desks.put(name, place);
    Удобство в том что поиск места по имени ученика заменяется на place = desks.get(name); И не требуется сложной логики в виде вложенных циклов.

    Но можно и хранить в обратном порядке аргументов desks.put(place, name);
    В этом случае легко проверять занято место или нет name = desks.get(place);

  2. Для хранения незанятых мест используйте контейнер-множество Set.

    В конструкторе инициализируйте это множество всеми возможными размещениями. Затем, при размещении ученика, можно брать первое попавшееся место.
    Выберите конкретную реализацию интерфейса Set которая хранит и возвращает элементы в том же порядке в котором они добавлялись в контейнер.

    Не забудьте удалить взятое размещение чтобы не использовать его повторно.

    Возможно потребуются поля класса для хранения размера класса rows и columns.
    Для проверки решения щелкните Check
